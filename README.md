# opc-hub

一个面向 OPC 生态的共建仓库。

这里不做单一官网，而是把 OPC 相关信息、全国站点、实用工具和公开资料整理成一个可持续协作的 GitHub 目录，方便更多人一起补充、校对和维护。

## 这个仓库要解决什么问题

- OPC 相关信息分散在微信群、朋友圈、活动页和个人笔记里，不方便搜索和沉淀
- 不同城市是否已有站点、页面是否公开、信息是否有效，缺少统一目录
- 新站点发起人想搭页面、做活动、收信息时，常常不知道从哪里开始

`opc-hub` 的目标不是替代每个站点自己的页面，而是做一个公开、可协作、可持续维护的索引仓库。

## 仓库结构

```text
.
├── .github/                  # Issue / PR 模板
├── sites/                    # 全国 OPC 站点目录
│   └── china/
├── tools/                    # 可复用工具、模板、清单
├── resources/                # 公开资料、方法、术语说明
├── templates/                # 新增站点 / 工具时使用的模板
└── docs/                     # 仓库使用说明与协作文档
```

## 当前收录方向

### 1. 全国站点目录

重点收录：

- 已公开的 OPC 站点主页
- 社区介绍页、报名页、活动页
- 能证明站点存在的公开链接
- 城市级线索入口

### 2. OPC 实用工具

重点收录：

- 站点收录字段模板
- 活动发起清单
- 站点页面结构模板
- FAQ 草案
- 城市站点比较维度

### 3. 资料与方法

重点收录：

- OPC 基础说明
- 常见问题整理
- 页面案例拆解
- 站点运营经验

## 如何参与共建

1. 先阅读 [CONTRIBUTING.md](./CONTRIBUTING.md)
2. 按模板新增或补充内容
3. 发起 Pull Request
4. 如果还没有完整资料，先用 Issue 提交线索

如果你不知道该从哪里开始，最简单的方式是：

- 提交一个新站点
- 修正某个站点的失效链接
- 补一个常用工具模板
- 把某个城市的公开线索整理出来

## 内容规范

- 优先收录可公开访问的信息
- 尽量写明来源链接
- 不收录明显无法核验的传闻
- 联系方式、地址等敏感信息以公开信息为准
- 如果信息不完整，可以标注为“待核验”而不是硬写结论

## 后续可扩展方向

- 按区域生成城市索引
- 给站点增加状态字段，例如“已收录 / 待核验 / 征集中”
- 为 GitHub Pages 准备自动生成页面
- 通过 GitHub Actions 校验目录字段完整性
