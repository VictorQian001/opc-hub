# 如何共建

[返回 Wiki 首页](./README.md)

`opc-hub` 采用“主仓库 + wiki 风格目录”的方式协作。

也就是说：

- `docs/` 负责可浏览的知识入口
- `sites/`、`resources/` 负责存放实际内容
- 所有人都通过 Issue 和 Pull Request 共建

## 最常见的三种贡献

### 1. 新增一个站点

适合你知道某个城市已经有公开站点、页面或活动入口。

先看：

- [站点目录](./sites/README.md)
- [城市入口](./cities/README.md)
- [站点模板](../templates/site-entry.md)

如果你是社区主理人，建议顺手补齐：

- 进入门槛
- 成本
- 报名方式
- 是否长期开放
- 优惠政策

### 2. 新增一个资源

适合你已经整理好了可复用的字段模板、活动清单、FAQ 草案，或者发现了值得收录的实用网站。

先看：

- [资源库](./resources/README.md)
- [资源模板](../templates/resource-entry.md)

### 3. 修正已有信息

适合你发现链接失效、状态过期、联系方式有误或描述不准确。

优先在 PR 中写明：

- 你改了什么
- 为什么改
- 依据是什么

## 提交原则

- 优先公开可核验信息
- 不写未经确认的结论
- 信息不完整时标注“待核验”
- 敏感信息只采用已公开内容
