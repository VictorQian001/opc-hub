# 状态定义

[返回资料目录](./README.md)

为了让目录更容易维护，站点条目建议统一使用以下状态。

## 已收录

表示这个站点已经有较明确的公开信息，至少具备以下条件中的一部分：

- 有公开页面
- 有明确站点名称
- 有基础说明
- 有可追踪来源

## 待核验

表示已经有线索，但还不适合写成明确结论。

常见情况：

- 只有零散提及，没有完整页面
- 找不到最新公开入口
- 主理人、城市或定位信息仍不确定

## 征集中

表示这个城市或方向已经预留目录位置，但当前还缺少合格的公开资料。

这种状态适合：

- 告诉协作者这里值得补充
- 让目录结构先长出来
- 避免每次都从零讨论“是否应该建这个页面”

## 站点页建议字段

为了让站点信息更有用，建议除了基础介绍外，再补以下字段：

- 面向人群
- 可提供内容
- 社区额外权益
- 公开入口
- 信息来源

其中“社区额外权益”优先写这个社区本身能提供、且不等同于城市通用政策的内容，例如：

- 入驻后的共享办公与活动空间
- 社区自带的工具支持、Tokens、智能体或数据服务
- 社群活动、资源对接、场景撮合
- 社区专属折扣、免租期或报名优先权

城市层面的扶持政策、政策文件和数据源链接，统一放在城市页维护。
